# 🍽️ 逢甲吃吃喝喝平台 — 功能分級

> 依據「純 HTML/CSS/JS 靜態原型」的能力範圍，將所有保留功能分為三類。

---

## A. ✅ 第一版一定要完成的核心功能

> 這些是讓原型能「跑起來、能展示」的最基本功能。

| # | 功能 | 實作方式 |
|---|------|---------|
| 1 | **搜尋與篩選功能** | JS 關鍵字搜尋 + 分類篩選（價格、類型） |
| 2 | **精準預算分類導航** | 銅板美食 / 學生正餐 / 聚餐首選 的分類標籤按鈕 |
| 3 | **店家資訊卡片展示** | 每家店顯示：名稱、價位、評分、照片、標籤 |
| 4 | **CP值量化篩選器** | 用星星 + CP值標籤排序，篩選高CP值店家 |
| 5 | **韭菜警報（黑名單）系統** | 店家卡片上顯示⚠️警告標籤 + 防詐提示頁面 |
| 6 | **聯絡/導航功能** | 點擊按鈕開啟 Google Maps 導航或店家社群連結 |
| 7 | **客製化飲食標籤** | 素食🌱、健身💪、過敏⚠️ 等篩選標籤 |
| 8 | **RWD 響應式設計** | 手機、電腦都能正常瀏覽 |

---

## B. 🎨 可以先做成展示畫面或模擬流程的功能

> 這些功能用**靜態假資料 + 互動動畫**模擬即可，展示時說明「未來會接後端」。

| # | 功能 | 模擬方式 |
|---|------|---------|
| 1 | **社群與評價功能** | 展示幾則寫死的假評論，有星星評分 UI |
| 2 | **今天吃什麼隨機轉盤** | JS 做轉盤動畫，從預設清單隨機抽取 |
| 3 | **動態營業狀態顯示** | 用假資料標示「營業中🟢 / 休息中🔴」 |
| 4 | **逢甲人認證機制** | 做出「學生認證」的 UI 表單畫面（不實際驗證） |
| 5 | **逢甲私房名單** | 獨立一個「在地推薦」專區，展示精選店家 |
| 6 | **即時優惠/惜食發布** | 展示幾個假的優惠卡片（如「晚8點後8折」） |
| 7 | **單人友善標註** | 店家卡片加上「🙋單人友善」標籤 |
| 8 | **抗漲地圖/趕課快餐標籤** | 加上「⚡5分鐘快取」標籤，做篩選按鈕 |
| 9 | **冷氣/插座/限時環境標記** | 店家卡片加上「❄️冷氣 🔌插座」小圖示 |
| 10 | **店家整潔度即時評分** | 用假資料顯示整潔度星星 |
| 11 | **評論審核機制** | 模擬「檢舉」按鈕 UI（點擊彈出提示框） |

---

## C. 🚫 暫時先不做的功能

> 這些需要**後端伺服器、資料庫、第三方 API**，純前端做不到或意義不大。

| # | 功能 | 不做的原因 |
|---|------|-----------|
| 1 | **擁擠度與步行預估** | 需要 Google Maps API + 即時人流數據 |
| 2 | **剩餘座位即時回報** | 需要後端 + 店家即時回報系統 |
| 3 | **逢甲周邊天氣與攤位連動** | 需串接天氣 API + 複雜邏輯 |
| 4 | **消費積點換食計畫** | 需完整的後端會員與積分系統 |
| 5 | **即時資訊與互動功能** | 需即時數據庫（如 Firebase） |
| 6 | **物價追蹤與漲價通報** | 需要長期數據收集與比對系統 |
| 7 | **權限管理** | 需後端帳號系統 |
| 8 | **資料備份 / 加密 / SQL防護 / 防暴力破解** | 這些是後端部署時的任務 |
| 9 | **高併發 / 負載平衡 / API響應** | 屬於伺服器架構層級 |
| 10 | **多語系支援 / 影像壓縮 / 模組化架構** | 未來版本再考慮 |
| 11 | **個資去識別化 / 隱私保護** | 第一版不收集真實個資，暫無此需求 |

---

## 📊 總結

| 分類 | 數量 | 說明 |
|------|------|------|
| **A. 核心功能** | 8 項 | 本週一定完成，讓原型有完整體驗 |
| **B. 展示/模擬** | 11 項 | 用假資料做出畫面，展示未來願景 |
| **C. 暫不做** | 11 項 | 需後端支援，留給未來版本 |

> **A + B 合計 19 項功能的畫面**，用純 HTML/CSS/JS 本週完全可以做完！
> 展示時可以說明 B 類功能「目前是模擬資料，未來會串接後端」，這在課堂展示中是完全被接受的。
